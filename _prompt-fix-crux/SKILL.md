# 提示词方法论化

修复提示词中的任何质量问题，遵循一条路径：**诊断根因 → 树立准则 → 最小化落笔**。

核心立场：**方法论驱动，而非就事论事**。

- 就事论事 = 看到症状就加一条规则
- 方法论驱动 = 先弄清底层原理，再以一条准则统摄所有症状

## 诊断根因

产出有问题时，先追问根因——不要被表面症状牵着走：

- 提示词确实没覆盖这个维度
- 提示词覆盖了但措辞方向有误、自相矛盾、或优先级不清
- 提示词说对了但在长上下文中被淹没（位置/权重问题）
- 根本不是提示词的问题（模型能力边界、上下文长度、任务本身的固有难度）

诊断的目的是找到「最小且正确的干预点」，而非机械对应到某个固定动作。有时正确答案是不改。

## 树立准则

确认需要改提示词时，拒绝「加规则」思维，转向「立准则」思维：

- **正面描述期望**（方法论），而非反面罗列禁令（就事论事）
- **一条准则统摄**（方法论），而非把观察到的每个症状都写成一条规则（就事论事）
- 症状是诊断线索，不是写进提示词的内容

判断标准：能否把 N 条具体规则抽象为 1 条通用准则？抽象不出来，说明根因还没找到。

## 最小化落笔

- 能不动就不动——现有措辞是否已隐含覆盖？
- 能改一句就不加一段
- 反例留在对话里，不写进提示词
- 修完自检：还能再砍吗？
