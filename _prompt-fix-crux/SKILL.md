# 提示词方法论化

修复提示词中的任何质量问题，严格按三步走：

> **1 诊断根因 → 2 树立准则 → 3 精准落笔**

核心立场：**方法论驱动，而非就事论事**。

- 就事论事 = 看到症状就加一条规则
- 方法论驱动 = 先弄清底层原理，再以一条准则统摄所有症状

---

## 1. 诊断根因

产出有问题时，先追问根因——不要被表面症状牵着走。逐一排查：

1. 提示词确实没覆盖这个维度
2. 提示词覆盖了但措辞方向有误、自相矛盾、或优先级不清
3. 提示词说对了但在长上下文中被淹没（位置/权重问题）
4. 根本不是提示词的问题（模型能力边界、上下文长度、任务固有难度）

诊断的目的是找到「最小且正确的干预点」，而非机械对应到某个固定动作。有时正确答案是不改。

---

## 2. 树立准则

确认需要改提示词时，拒绝「加规则」思维，转向「立准则」思维：

1. **正面描述期望**（方法论），而非反面罗列禁令（就事论事）
2. **一条准则统摄**（方法论），而非把观察到的每个症状都写成一条规则（就事论事）
3. 症状是诊断线索，不是写进提示词的内容

判断标准：能否把 N 条具体规则抽象为 1 条通用准则？抽象不出来，说明根因还没找到。

---

## 3. 精准落笔

基于根因的修改，落笔位置和方式都要重新评估——不是“哪冒泡改哪”：

1. **改对位置**——暴露症状的地方未必是病灶；顺因果链追到真正该动的位置，可能要改的是上游而非表面
2. **改出最优解**——允许合并多处改动、调换位置、重写表达方式；只要最终效果更好，不局限于原位替换
3. **保持 token 克制**——不追求“改得少”，也不强求改后总量更短；确需新增约束或定义时大方加，但每一笔都要有明确理由，不添无效冗余
4. 反例留在对话里，不写进提示词
5. 修完自检：有没有“治了症状却错过病因”的改动？每一处新增/删除/改写是否都有充分理由？
