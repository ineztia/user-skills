# 目标

将单体 AGENTS.md / Claude.md 拆分为：精简入口文件 + 按职责分层的模块目录。入口只做路由与加载，具体指令由各模块承载。

# 工作流程

## 1. 备份原文件

1.1 在执行任何修改前，备份原始文件
1.2 备份命名：`{原文件名}.{unix时间戳}.bak`（如 `AGENTS.md.1711089600.bak`）
1.3 备份文件与原文件位于同一目录

## 2. 分析原文件

2.1 通读原文件，识别所有指令段落
2.2 按职责归类，标记每段属于哪个层级（见下方分层原则）
2.3 识别段落间的依赖关系和加载优先级

## 3. 设计拆分方案

3.1 根据分层原则确定模块目录结构（目录名、文件名根据实际内容动态命名）
3.2 确定入口文件的路由逻辑（哪些模块始终加载，哪些按需引用）
3.3 处理跨模块依赖：提取共享定义到公共模块，或在入口中声明

## 4. 执行拆分

4.1 逐模块提取内容，保持原始语义不变
4.2 每个模块文件应自包含、职责单一
4.3 生成入口文件：只包含角色一句话定义、模块加载指令、全局约束（如有）

## 5. 验证

5.1 检查：原文件所有指令是否已被覆盖，无遗漏
5.2 检查：入口文件是否足够精简，无冗余内容
5.3 检查：模块间无循环依赖、无重复定义

# 分层原则

按以下层级组织模块（层级名称和具体拆分粒度应根据原文件内容动态调整）：

| 层级 | 职责 | 示例 |
|------|------|------|
| 身份层 | 角色设定、能力边界 | persona、boundaries |
| 准则层 | 思维框架、沟通风格、质量标准 | thinking、communication |
| 工作流层 | 各类任务的执行流程 | research、review、coding |
| 领域层 | 项目上下文、术语表 | project-context、glossary |
| 工具层 | 工具使用规范 | file-ops、search、mcp |

# 约束

1. 入口文件行数应控制在 200 行以内
2. 单个模块文件只承载一个职责
3. 模块文件名使用 kebab-case，语义自明
